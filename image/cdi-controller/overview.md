## About CDI Controller

The CDI Controller is a component of the
[Containerized Data Importer (CDI)](https://github.com/kubevirt/containerized-data-importer) project, which is part of
the [KubeVirt](https://kubevirt.io/) ecosystem.

CDI provides a Kubernetes extension that enables importing, uploading, and cloning of persistent volume claims (PVCs).
The cdi-controller component manages the lifecycle of these data operations, coordinating the import, upload, and clone
workflows within a Kubernetes cluster.

For more details, see https://github.com/kubevirt/containerized-data-importer.

## About Docker Hardened Images

Docker Hardened Images are built to meet the highest security and compliance standards. They provide a trusted
foundation for containerized workloads by incorporating security best practices from the start.

### Why use Docker Hardened Images?

These images are published with zero-known CVEs, include signed provenance, and come with a complete Software Bill of
Materials (SBOM) and VEX metadata. They're designed to secure your software supply chain while fitting seamlessly into
existing Docker workflows.
